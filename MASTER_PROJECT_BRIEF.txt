================================================================================
SECTION 4 — AI & RAG PIPELINE ANALYSIS
================================================================================
Process: Message -> api/chat -> lookup data sources -> searchKnowledge() -> Groq.
FORENSIC AUDIT OF lib/rag.ts:
1. Fetches all text documents for agent from Mongo.
2. Splits every text document in runtime using regex.
3. For every single chunk, executes a network fetch to Hugging Face API to generate an embedding.
4. Performs Array.reduce Cosine Similarity in raw JS memory.
5. Sorts and returns Top-3 to inject into prompt.
SCALABILITY RATING: FATAL. 100 chunks would trigger 100 sequential external HTTP requests per single user chat message, timing out the gateway.

SECTION 5 — DATA INGESTION SYSTEM

================================================================================
SECTION 6 — MULTI-TENANCY & SECURITY AUDIT
================================================================================
- Isolation: Clean workspaceId and agentId scoping guards queries correctly.
- Widget Security: Explicit domain Referer checks limit third-party hijacking.
- Vulnerabilities: 
  - RateLimiter throttles Mongo: High traffic would cause write locks and kill the app.
  - Information Leakage: GET /api/agents dumps the full object including potentially private properties to the client.

================================================================================
SECTION 7 — CREDITS & BILLING SYSTEM
================================================================================
- Status: High Quality.
- Decrement Logic: Uses atomic Mongo $cond increment commands to ensure users cannot overdraw single free credits across parallel requests.
- Webhooks: Correctly constructed via Dodo SDK with secret verification.

================================================================================
SECTION 8 — DEPLOYMENT & WIDGET SYSTEM
================================================================================

- Friction: The instructions copy hardcoded 'agentdesk.ai' addresses which breaks dev-mode testing workflows unless manual rewrites happen.

SECTION 12 — BUSINESS MODEL

================================================================================
SECTION 13 — ROADMAP RECOMMENDATIONS
================================================================================
PHASE 1 (SURGERY):
- Activate pdf-parse imports in datasources route.
- Integrate a vector persistence layer (Mongo Atlas Vector or Pinecone).
- Save embeddings at upload-time, not at chat-time.
PHASE 2 (STABILITY):
- Shift rate limiting off MongoDB and onto Upstash/Redis.
- Implement true LLM streaming via Vercel AI SDK.
PHASE 3 (ENTERPRISE):
- Full human agent chat takeover / Live Inbox implementation.
- Implement proper web crawlers via firecrawl/cheerio.
