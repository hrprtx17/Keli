1. EXECUTIVE OVERVIEW (THE ELEVATOR PITCH)
--------------------------------------------------------------------------------
* Tagline:
  "Custom AI Support Agents trained on your data. Deployed in minutes."

* Mission:
  Keli AI bridges the gap between autonomous customer support and human agent
  escalations. It allows businesses to train bespoke AI chat widgets on their
  knowledge base (documents, links, crawls), resolve up to 85% of queries
  instantly, and seamlessly transfer complex or high-intent leads to human 
  representatives in a gorgeous, glassmorphic central dashboard.

* Target Audience:
  - SaaS & Web Applications seeking to reduce customer support overhead.
  - E-commerce stores needing 24/7 checkout and order status guidance.
  - Digital Agencies looking to offer premium custom chat widgets to clients.

* Live Deployment:
  - Production URL: https://keliai.netlify.app
  - Hosted on Netlify with @netlify/plugin-nextjs

2. BRAND DESIGN SYSTEM (FONTS & COLOR SCHEMES)
--------------------------------------------------------------------------------
Keli AI features an elite, modern typography pairing and a sleek dual-mode
(light + dark) curated palette optimized for dynamic web presentation.

* Typography Pairing:
  1. Headings & High-Impact Numbers: "Space Grotesk" (Google Fonts)
     - Rationale: A geometric, slightly futuristic sans-serif that exudes 
       high-end boutique tech aesthetics. Gives numbers and titles a crisp,
       authoritative, and modern feel.
     - CSS Variable: --font-heading (--font-space-grotesk)
  2. Body, Labels & Inputs: "Outfit" (Google Fonts)
     - Rationale: A premium, ultra-readable, round geometric typeface designed 
       specifically for fast legibility, modern SaaS dashboard cards, and 
       interactive web forms.
     - CSS Variables: --font-sans, --font-jakarta (both map to --font-outfit)

* Primary Color Palette:
  - Core Brand Accent (Orange-Amber): HSL(25, 95%, 53%) / Approx Hex #F97316
    - Rationale: Represents active support, warmth, energy, and high conversion.
    - Used for: Primary buttons, CTA rings, sidebar active states, chart accents.
  - Pure Grounding Dark Accent: HSL(0, 0%, 4%) for dark mode background
    - Rationale: Provides sophisticated, premium structure in dark mode.
  - Cool Status Accents:
    - In-Progress Blue: Hex #3b82f6 (Tailwind Blue-500)
    - Resolved Emerald: Hex #10b981 (Tailwind Emerald-600)
    - Gray Muted Text: HSL(0, 0%, 45%) light / HSL(0, 0%, 50%) dark
    - Soft Dashboard Borders: HSL(0, 0%, 90%) light / HSL(0, 0%, 14%) dark

* Dark Mode:
  - Full dark theme support via next-themes ThemeProviderWrapper.
  - All CSS tokens defined in both :root (light) and .dark (dark) modes.
  - Cards shift to HSL(0, 0%, 8%) backgrounds with 97% white foregrounds.

3. HIGH-PERFORMANCE TECH STACK
--------------------------------------------------------------------------------
Keli AI is built on a modern, robust framework tailored for sub-100ms response
times and pristine type-safety:

* Core Framework: Next.js 16 (App Router with dynamic server-side routing)
* Rendering & Bundle Optimization: React 19 (latest concurrent features)
* Client-Side State: TanStack React Query v5 for instant cache sync
* Global State: Zustand v5 (lightweight store management)
* Styling & Transitions: Tailwind CSS v4, tw-animate-css, & Framer Motion v12
* UI Component Library: shadcn/ui (Radix UI primitives + CVA)
* Authentication: NextAuth.js v5 beta (Google OAuth + Email/Password Credentials)
* Database: MongoDB Atlas with Mongoose v9 ODM
* AI Inference: Groq SDK + Vercel AI SDK v6 (ai + @ai-sdk/groq)
* Vector Embeddings: HuggingFace Inference API (sentence-transformers/all-MiniLM-L6-v2)
* Vector Search: MongoDB Atlas $vectorSearch aggregation pipeline
* Text Splitting: LangChain RecursiveCharacterTextSplitter
* Web Crawling: Puppeteer v24 (headless Chrome) + Cheerio v1 (HTML parser)
* Email Notifications: Resend API (branded HTML ticket & human escalation emails)
* Payments & Billing: Dodo Payments (subscription management, webhook processing)
* Toast Notifications: Sonner v2 (premium micro-animations)
* Form Validation: React Hook Form + Zod v4
* Charts & Analytics: Recharts v3 (AreaChart, responsive containers)
* Date Utilities: date-fns v4
* File Parsing: pdf-parse v2 (PDFs), mammoth v1 (DOCX)
* Security: bcryptjs (password hashing), rate limiting with SecurityLog auditing
* Icons: Lucide React v1

4. DATABASE MODELS (MONGODB SCHEMAS)
--------------------------------------------------------------------------------
Keli AI uses 10 core Mongoose schemas:

  1. User         — name, email, password (bcrypt), role, workspaceId
  2. Workspace    — name, slug, ownerId, plan (free/premium), usage tracking
                    (monthlyCredits, creditsUsedThisMonth, addonCredits, resetDate),
                    limits (maxAgents, maxStorage), dodoSubscriptionId, dodoCustomerId
  3. Agent        — workspaceId, name, description, avatar, systemPrompt,
                    notificationEmail, model (default: llama-3.1-8b-instant),
                    config (temperature, maxTokens, language, tone, role, guidelines,
                    linkUsage), widgetConfig (primaryColor, position, welcomeMessage,
                    placeholder, showBranding), apiKey (unique), isActive,
                    totalConversations, totalMessages
  4. Conversation — workspaceId, agentId, visitorId, metadata
  5. Message      — conversationId, role, content, timestamps
  6. Ticket       — workspaceId, agentId, conversationId, title, description,
                    customerEmail, customerName, status (open/in-progress/resolved/closed),
                    priority (low/medium/high/urgent), assignedTo, replies[]
  7. DataSource   — agentId, workspaceId, type (pdf/docx/txt/csv/md/url/text/faq),
                    name, content, status (pending/processing/ready/error),
                    metadata (size, pages, url, wordCount)
  8. KnowledgeChunk — agentId, workspaceId, dataSourceId, text, embedding (number[]),
                    metadata (mixed)
  9. FAQ          — agentId, workspaceId, question, answer, category, isActive
  10. SecurityLog — eventType (rate_limit/auth_failure/unauthorized_domain/
                    credit_exhausted/bot_detected/webhook_failure), ipAddress,
                    userAgent, endpoint, details, workspaceId
                    (auto-expires after 30 days via TTL index)

5. HOW IT WORKS (THE CUSTOMER JOURNEY FLOW)
--------------------------------------------------------------------------------
[Step 1: Onboard & Source Data]
Business owners paste website links or upload training manuals (PDF, DOCX, TXT,
CSV, Markdown) into Keli AI. The system runs a deep Puppeteer-powered headless
Chrome crawl to parse and structure the knowledge base. Content is split into 
chunks via LangChain's RecursiveCharacterTextSplitter and embedded using 
HuggingFace's all-MiniLM-L6-v2 model.

[Step 2: Deploy Custom AI Widget]
An elegant, tiny JavaScript script (widget.js / keli.js) code block is copied
and pasted into the client's website. The widget loads as a cross-origin iframe
from keliai.netlify.app with configurable colors, greetings, and branding.
CORS headers are configured in netlify.toml for seamless external embedding.

[Step 3: Autonomous AI Handling]
A website visitor chats with the widget. The Keli AI agent responds instantly
using Groq inference (Llama 3.1-8B or configurable model). Relevant knowledge
chunks are retrieved via MongoDB Atlas $vectorSearch with cosine similarity,
injected as RAG context into the system prompt.

[Step 4: Smart Human Escalation]
If the visitor requests human assistance or exhibits high-intent purchase 
behavior, the AI instantly creates a Ticket and sends a branded HTML email
notification via Resend to the agent's configured notification email.

[Step 5: Live Human Dashboard Review]
The business team logs in, views the continuous chat timeline history, inspects
the auto-generated AI conversation summary, saves internal notes, resolves the
ticket, or emails the visitor back. All from a premium glassmorphic dashboard.

6. KEY PRODUCT FEATURES
--------------------------------------------------------------------------------
* 📊 Insights Dashboard:
  An analytics-first home screen with animated stat cards (conversation volume,
  escalated tickets, credit utilisation), secondary KPI row (response speed,
  resolution clearance, visitor CSAT), and a responsive Recharts AreaChart
  showing temporal interaction flow across a 7-day calendar.
  Gradient background glows and Framer Motion hover animations on cards.

* 📋 Tickets Panel:
  An ultra-modern dual-pane dashboard containing glassmorphic cards, pulsing status
  LED indicators, custom priority weights (urgent, high, medium, low), and 
  timeline-style messenger feeds. Supports inline replies.

* 📥 Unified Inbox:
  A clean center where support representatives can see active live chat queues 
  and switch dynamically between client chats and a sandbox AI Playground.

* 🧠 Knowledge Base (Training Data):
  Upload raw text, markdown documents, PDFs, DOCX files, scrape single URLs, or
  initiate recursively nested website crawls (up to 15 pages). The AI instantly
  processes the corpus into vector-embedded chunks. Supports FAQ management with
  categorised question-answer pairs.

* 🚀 Deploy Portal:
  Easy-to-copy HTML script tags (widget.js / keli.js), dynamic API endpoints,
  and real-time custom configuration widgets to modify button styles, greeting
  welcome messages, positions, and colors. Live preview on a test page
  (widget-test.html).

* 💰 Billing & Financial Overview:
  Capacity lifecycle tracking with animated credit counters, operational load
  progress bars, system invoices table, payment gateway display (Visa card
  vault), and active plan status. Powered by Dodo Payments integration.

* 📈 Usage Analytics:
  Detailed usage page with comprehensive credit consumption tracking, addon
  credit management, and monthly reset cycle monitoring.

* 💎 Plans & Pricing:
  Full pricing page with tier comparison matrix. Free plan (500 credits/month,
  1 agent, 10 max storage) vs Premium Pro plan ($29/month with expanded limits).
  Upgrade flow via Dodo Payments checkout.

* 🛡️ Multi-Agent Identity & Settings:
  Configure unique behavioral prompt systems, customize avatar images, select 
  AI models, set temperature/maxTokens, configure language/tone/role/guidelines,
  manage notification emails, and safely delete active agents.
  Settings page for workspace profile, domain whitelisting, and account management.

* 🔐 Security & Rate Limiting:
  MongoDB-backed rate limiting with atomic token decrement. SecurityLog model
  tracks abuse events (rate limits, auth failures, unauthorized domains, credit
  exhaustion, bot detection, webhook failures). Logs auto-expire after 30 days.

* 📧 Email Notification System:
  Two branded HTML email templates via Resend API:
  1. New Support Ticket — sent when AI creates a ticket.
  2. Human Support Request — sent when visitor requests live human agent.
  Both include deep links directly into the dashboard inbox.

* 🔒 Authentication:
  NextAuth.js v5 with dual providers: Google OAuth and Email/Password credentials.
  Rate-limited login attempts (5 per 60 seconds). Protected routes enforced via
  middleware (dashboard, agents, settings, onboarding).

* 🌓 Dark Mode:
  Full dark mode support via next-themes. Toggle available in sidebar.
  All components, cards, and charts respect theme tokens.

* 📄 Legal Pages:
  Privacy Policy and Terms of Service pages.

* 💬 Conversations Manager:
  View and manage all visitor conversations across agents.

8. INFRASTRUCTURE & DEPLOYMENT
--------------------------------------------------------------------------------
* Hosting: Netlify (with @netlify/plugin-nextjs v5)
* Build Command: npm run build
* Publish Directory: .next
* Node Version: >=20.9.0
* CORS Configuration: netlify.toml headers for widget.js, keli.js, widget/*,
  widget-test.html, /api/widget-chat, /api/agents/public/*
* Security Headers: X-Content-Type-Options, X-XSS-Protection,
  Referrer-Policy, Permissions-Policy (camera, microphone, geolocation disabled)
* Middleware Bypass: Static assets (widget.js, keli.js, widget-test.html,
  favicon, icons) and /api/* routes bypass auth middleware.
